# Java Core Concepts

A structured collection of core Java concepts with practical examples, focusing on topics that are commonly confusing or misunderstood by developers.

---

## About This Repository

This repository is built to help developers understand Java concepts that are often unclear or poorly explained elsewhere. Each topic includes clean code examples, explanations, and common mistakes to avoid.

The repository is open for contributions. Anyone can contribute by following the guidelines below.

---

## Topics Covered

This repository currently focuses on confusing and frequently misunderstood Java concepts. More topics will be added over time.

Some areas covered include:

- Object-Oriented Programming (OOP) — inheritance, polymorphism, abstraction, encapsulation
- Collections Framework — List, Set, Map, Queue and their internal working
- Exception Handling — checked vs unchecked, custom exceptions, best practices
- Multithreading — threads, synchronization, deadlock, executor service
- Streams and Lambda — functional interfaces, method references, stream operations
- Memory Management — stack vs heap, garbage collection, object lifecycle
- Design Patterns — creational, structural, behavioral patterns with real examples
- Java Internals — String pool, autoboxing, static keyword, final keyword

## How to Contribute

Contributions are welcome from everyone. Please read the steps below before submitting.

### Step 1 — Fork the Repository

Click the "Fork" button on the top right of this page to create your own copy of this repository.

### Step 2 — Clone Your Fork

```bash
git clone https://github.com/your-username/java-core-concepts.git
cd java-core-concepts
```

### Step 3 — Switch to the Contribute Branch

```bash
git checkout contribute
```

### Step 4 — Make Your Changes

Add your code, fix an explanation, or improve an existing example. Keep your changes focused on one topic at a time.

### Step 5 — Commit Your Changes

```bash
git add .
git commit -m "Add: explanation for XYZ concept"
```

### Step 6 — Push to Your Fork

```bash
git push origin contribute
```

### Step 7 — Open a Pull Request

Go to the original repository and open a Pull Request from your fork's `contribute` branch to the `contribute` branch of this repository.

Your PR will be reviewed by the maintainer. If everything looks good, it will be merged.

---

## Contribution Guidelines

- Write clean and readable code with proper comments.
- Each concept should have at least one working example.
- Do not submit copied content from other sources.
- Keep variable and method names meaningful.
- Follow standard Java naming conventions.
- One topic per Pull Request is preferred.

---

## Requirements

- Java 8 or above
- Any IDE (IntelliJ IDEA, Eclipse, VS Code)
- Basic understanding of the topic you are contributing to
